## Get started in three steps

Spin up your LoneScale docs locally, connect deployments, and ship your first change.

### Step 1: Set up locally

<AccordionGroup>
  <Accordion icon="copy" title="Clone your docs">
    Your documentation lives in a GitHub repository. You can find or change the connected repository from your LoneScale Docs dashboard.

    1. Open your docs repo from your dashboard.
    2. Follow GitHub’s guide to clone it locally: <a href="https://docs.github.com/en/repositories/creating-and-managing-repositories/cloning-a-repository" target="_blank" rel="noopener noreferrer">Cloning a repository</a>.

  </Accordion>

  <Accordion icon="rectangle-terminal" title="Preview your docs">
    1. Install the Mintlify CLI: `npm i -g mint`  
    2. From the repo root, run: `mint dev`  
    3. Open `http://localhost:3000` to see your docs live.

    <Tip>Your preview auto-reloads as you edit files.</Tip>

  </Accordion>

  <Accordion icon="key" title="Configure API Playground auth (x-api-key)">
    If you’re exposing the LoneScale API in the Playground, add an API key security scheme to your OpenAPI file.

```yaml
openapi: 3.1.0
components:
  securitySchemes:
    ApiKeyAuth:
      type: apiKey
      in: header
      name: x-api-key
security:
  - ApiKeyAuth: []
```

Clients will send: `x-api-key: YOUR_API_KEY`.

**Common errors**

- `401 Invalid API Key`: key missing or incorrect.
- Check your environment/organization API key from the LoneScale app.
    </Accordion>
</AccordionGroup>

### Step 2: Deploy your changes

<AccordionGroup>
  <Accordion icon="github" title="Connect GitHub for auto-deploys">
    Install the Mintlify GitHub app from your docs dashboard. Every push to your default branch will automatically deploy your docs — no CI to manage.
  </Accordion>

  <Accordion icon="palette" title="Brand your LoneScale docs">
    1. Open `docs.json`.  
    2. Set `"name"` to `"LoneScale Docs"` (or your project name).  
    3. Update `"colors"` to match your brand.  
    4. Save and check `http://localhost:3000`.

    <Tip>Try changing the primary color to quickly validate your palette.</Tip>

  </Accordion>
</AccordionGroup>

### Step 3: Go live

<Accordion icon="rocket" title="Publish">
  1. Commit and push to your main branch. 2. Your docs deploy automatically and
  are live within moments.
</Accordion>
